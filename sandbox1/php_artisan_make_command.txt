# 1. Logic / Events / Queue
# システムの中核となる振る舞いを分離する機能です。

    app/Events
        コマンド: php artisan make:event [Name]
        用途: システム内での「出来事」を定義、Listenersが使うためのデータを定義し、データを受け取って保持する
        例：Event: OrderCompleted(オーダー完了)
            ❌ メールの送信処理を書く
            ❌ データベースの更新処理を書く
            ❌ 外部APIを叩く
            これらはListeners側の処理

    app/Listeners
        コマンド: php artisan make:listener [Name]
        用途: イベント発生時に実行する処理を定義。
        例：Listener 1: SendInvoiceMail (領収書送付)
            Listener 2: ReduceInventory (在庫減らす)
            Listener 3: AddSalesBonus (営業成績に加算)

    app/Jobs
        コマンド: php artisan make:job [Name]
        用途: キューでバックグラウンド実行する重い処理
              Commandとの使い分けとしては
              Command: 主に開発者やシステム（OSのCron）が手動または定期的に実行するもの
              Job: 主にアプリケーションの処理中に、ユーザーを待たせないために「バックグラウンド(非同期)」の「タスク単位」
                   DBは使うが、結果をユーザーが即座に知る必要がないもの」は、積極的に非同期化すべき

    app/Observers
        コマンド: php artisan make:observer [Name]
        用途: モデルの保存や更新などの監視と自動処理

# 2. HTTP / API / Validation
# リクエスト制御や入力チェックを行う機能です。

    app/Http/Middleware
        コマンド: php artisan make:middleware [Name]
        用途: リクエストの前後に挟む共通フィルター

    app/Http/Requests
        コマンド: php artisan make:request [Name]
        用途: バリデーションと認可ロジックの分離

    app/Http/Resources
        コマンド: php artisan make:resource [Name]
        用途: APIレスポンスのJSON整形

    app/Rules
        コマンド: php artisan make:rule [Name]
        用途: 独自のバリデーションルールの作成

# 3. Mail / Notifications / UI
# ユーザーへの出力や通知、画面部品を制御する機能です。

    app/Mail
        コマンド: php artisan make:mail [Name]
        用途: メールの構成やデザインの定義

    app/Notifications
        コマンド: php artisan make:notification [Name]
        用途: Slack、DB、メール等への一斉通知

    app/View/Components
        コマンド: php artisan make:component [Name]
        用途: Bladeコンポーネントのロジック部分

# 4. Security / Customization
# 高度なカスタマイズやセキュリティ設定を行う機能です。

    app/Policies
        コマンド: php artisan make:policy [Name]
        用途: モデル操作に対するユーザー権限の判定

    app/Exceptions
        コマンド: php artisan make:exception [Name]
        用途: アプリ独自のカスタムエラーハンドリング

    app/Casts
        コマンド: php artisan make:cast [Name]
        用途: モデルの値を独自型に変換するロジック

    app/Broadcasting
        コマンド: php artisan make:channel [Name]
        用途: WebSocket通信の認可チャンネル

    app/Console/Commands
        コマンド: php artisan make:command [Name]
        用途: 独自の artisan コマンドを作成

# 5. Manual Directories (Reference)
# 公式コマンドはないが、慣習的に手動で作成されるディレクトリです。

    app/Services
        用途: コントローラに書けない複雑なビジネスロジック

    app/Repositories
        用途: データベース操作の抽象化と共通化

    app/DTOs
        用途: データの受け渡しを構造化するクラス
